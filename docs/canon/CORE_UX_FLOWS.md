# PRAVADO v2 — CORE UX FLOWS
Version: v1.0 (Canon)

## FLOW 1 — Onboarding → Plan Lock
- AI gathers user inputs + performs research
- Produces initial plan for approval
- Once approved: plan is baseline; changes create revisions

## FLOW 2 — Daily Command Center Loop
- User sees proposals/tasks with why + confidence + impact
- Review via drawer/peek
- Approve/schedule → calendar item created
- Execution produces logs + outcomes

## FLOW 3 — PR Workflow
- Filter/find journalists
- View profile + activity/freshness/openness
- Generate pitch drafts (copilot)
- Approve outreach (confirm/hold-to-confirm as required)
- Track responses/coverage → feeds analytics

## FLOW 4 — Content Workflow
- Create briefs from SAGE signals
- Draft with copilot
- Approve + schedule
- Performance loops back into proposals

## FLOW 5 — SEO Workflow
- Scan/find issues/opportunities
- Approve fixes/tasks
- Schedule via calendar
- Results reflected in analytics/exposure

## FLOW 6 — Plan Change / Pivot
- User edits plan
- System generates revision proposal
- Approval creates new plan version, updates schedule intelligently

## FLOW 7 — Automation Mode Control
- Manual / Copilot / Autopilot per pillar
- Autopilot guarded with scope, caps, approvals, kill switch

## Compliance Checklist
- [ ] Every flow ends in an execution item or resolution
- [ ] No dead-end drilldowns
- [ ] Mode is always visible per action
